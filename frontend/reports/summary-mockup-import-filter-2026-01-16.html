const [variants, importedProductIds] = await Promise.all([
  getVariants(),
  getImportedProductIds()
]);

const filteredVariants = variants.filter(variant =>
  importedProductIds.includes(variant.product_id)
);

// Log for debugging
console.log('Filtered variants:', filteredVariants.length);
Rationale: Parallel fetching improves performance. Filtering ensures only imported products appear in the dropdown, matching admin dashboard behavior.

Q
Why use getImportedProductIds() instead of a direct database query?
The function encapsulates the logic for identifying imported products, ensuring consistency across the application. If the definition of "imported" changes, only one function needs updating.
Q
Will this affect performance?
Minimal impact. Both queries run in parallel using Promise.all(), and the filtering is done client-side on a small dataset. The getImportedProductIds() function is already optimized.
Q
What happens if no products are imported?
The dropdown will be empty. The console log will show "Filtered variants: 0". This is expected behavior and matches the admin dashboard which would also show no products.
